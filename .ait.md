# Conventional Commits for AIT

## Scope

Please observe the following rules for scopes:

- `systemd`: When referring to systemd services.
- `podman`: When referring to Podman quadlet files.
- `dockerfile`: When referring to the Dockerfile or the scripts included therein.
- `ci`: When referring to changes to workflows.
- `debian`: When referring to changes to Debian packaging.
- `debs`: When only dependencies are being updated.

Otherwise, do not use a scope unless it makes sense to do so!
